## Bugs corregidos esta sesión

### 1. Crear nueva asamblea sobreescribía la anterior
- **Causa**: `saveAssemblySettings` (PUT) hacía upsert sobre `findLatestAssemblyByProperty`,
  que retornaba la asamblea cerrada más reciente por `updatedAt`.
- **Fix**: Nuevo endpoint `POST /api/v1/properties/:propertyId/assemblies` que **siempre
  crea**, con guard 409 si ya hay una activa.
- Frontend: `createNewAssembly()` en `api.ts`, usado por `AssemblyListPage`.

### 2. `findLatestAssemblyByProperty` retornaba asambleas cerradas
- **Fix**: Filtra `status NOT IN ('closed', 'archived')`.
- Afecta todos los endpoints: agenda, convocatoria, acceso, readiness, avance de estado.
- `findAnyAssemblyByProperty` conservado para lecturas históricas sin filtro.

### 3. Hub cargaba datos de la asamblea equivocada
- **Fix**: Hub usa `listAssemblies(prop.id)` para encontrar la asamblea por `assemblyId`
  de la URL, en lugar de `fetchAssemblySettings` (que devolvía la "última activa").
- Vista de solo lectura (`isReadOnly`) para asambleas cerradas/archivadas.

### 4. Dashboard mostraba pasos completados y cronómetro de asamblea cerrada
- Solucionado automáticamente por fix #2.

### 5. Límite de cuerpo 2MB impedía guardar documentos adjuntos
- **Fix**: `MAX_BODY_BYTES` subido de 2MB a **20MB** en `apps/api/src/routes.ts`.

## Sala en vivo — implementación completa

### Canvas 16:9 full-width
- Proporción exacta `aspectRatio: "16/9"`, fondo degradado índigo.
- Contenido del canvas (3 modos controlados por `canvasMode`):
  - `"slide"` → muestra la diapositiva del punto activo
  - `"vote_active"` → muestra pregunta + cronómetro gigante (visible a todos)
  - `"vote_results"` → muestra gráfica de dona + veredicto APROBADO/NO APROBADO
- PiP cámara fijo en esquina inferior derecha (placeholder "Cámara pronto" hasta SDK de video).
- Cuando se comparte pantalla: el video de `getDisplayMedia` reemplaza el canvas.
- Badge "PANTALLA COMPARTIDA" / "RESULTADOS EN PANTALLA" según modo.

### Pantalla compartida
- `navigator.mediaDevices.getDisplayMedia({ video: true, audio: true })`
- `screenVideoRef` (useRef<HTMLVideoElement>) conectado por `useEffect` al `MediaStream`.
- Handler `ended` para limpiar state cuando el usuario detiene desde el navegador.

### Módulo de votación
**Flujo:**
1. Admin → botón "Iniciar votación" (pregunta prellenada del `votePrompt` del punto)
2. Modal setup: pregunta, regla (simple/dos tercios/unanimidad/informativa), duración (1-15 min)
3. Timer `useEffect` cada 1s → actualiza `voteCountdown` → canvas pasa a `"vote_active"`
4. Al terminar el tiempo (o "Cerrar votación"): modal de registro de resultados
5. Admin ingresa porcentajes: sí / no / abstención / blanco / total coeficiente
6. Previsualización en tiempo real con gráfica de dona (Recharts)
7. `saveVoteResult()` → calcula `approved` en backend → guarda en `AssemblyVoteResult`
8. Canvas pasa a `"vote_results"` con los resultados visibles a todos

**Revisitar votación:** píldoras de votaciones pasadas bajo la barra de controles;
al hacer click se muestra esa votación en el canvas.

**"Volver a diapositiva"** dismiss los resultados del canvas.

### Recursos adjuntos por punto
- `liveItemDocs` filtra `savedDocuments` por `agendaItemId` del punto activo
- Imágenes y PDFs: abren en visor inline (modal con `<img>` o `<iframe>`)
- Otros formatos: descarga directa + sugerencia de usar compartir pantalla

### Controles
Barra horizontal debajo del canvas (no columna lateral):
- Navegación anterior/siguiente + botones numerados por punto
- Compartir pantalla / Detener pantalla
- Iniciar votación / Cerrar votación (con cronómetro visible)
- Volver a diapositiva (cuando hay resultados en canvas)

## Backend — nuevas tablas y endpoints

### `AssemblyVoteResult` (Prisma + Supabase)
```
id, assemblyId, agendaItemId?, question, votingRule,
yesVotes, noVotes, abstainVotes, blankVotes, totalCoefficient,
approved, closedAt, createdAt
```

### Endpoints nuevos
- `POST /api/v1/properties/:propertyId/assemblies` — crear nueva asamblea (siempre nueva)
- `GET /api/v1/properties/:propertyId/assembly-votes` — lista votos asamblea activa
- `POST /api/v1/properties/:propertyId/assembly-votes` — registrar resultado de votación
- `GET /api/v1/properties/:propertyId/assemblies/:assemblyId/votes` — votos por ID
  (funciona para asambleas cerradas, para vista histórica)

### `computeVoteApproval()` en routes.ts
```
simple: yes > no
dos_tercios: yes / (yes+no+abstain) >= 2/3
unanimidad: yes === (yes+no+abstain) && yes > 0
ninguna: siempre false (informativa)
```

## Frontend `api.ts` — funciones nuevas
- `createNewAssembly(propertyId, input)` — POST (crear, no upsert)
- `listVoteResults(propertyId)` — GET votos asamblea activa
- `saveVoteResult(propertyId, input)` — POST guardar votación
- `listAssemblyVoteResults(propertyId, assemblyId)` — GET votos por ID (historial)

## Vista de asamblea cerrada (isReadOnly)
Cuando la URL apunta a una asamblea cerrada:
- Se muestra tarjeta de resumen (título, fecha, tipo, modalidad, base de votación, notas)
- Se cargan y muestran las votaciones con gráficas de dona y veredicto
- No se muestran las pestañas de edición
